Update the deployment wizard in the "LLM Bot Builder" project so that the Webhook URL field auto-populates fully based on the platform selected (Telegram, WhatsApp, Facebook, or Instagram) without requiring any manual typing.

1. **Backend**
   - In app.py, create a new route:
     ```python
     from flask import request

     @app.route('/api/webhook-url/<platform>', methods=['GET'])
     def get_webhook_url(platform):
         base_url = request.url_root.rstrip('/')
         return {"webhook_url": f"{base_url}/webhook/{platform}"}
     ```
   - This route should detect the current public server URL automatically and append the correct platform-specific path.

2. **Frontend**
   - In the deployment wizard UI (dashboard.js):
     * When the user selects a platform (Telegram, WhatsApp, Facebook, Instagram):
         - Call `/api/webhook-url/<platform>`.
         - Take the returned value and auto-fill the Webhook URL input field.
     - Example function:
       ```javascript
       async function autoFillWebhook(platform) {
           const res = await fetch(`/api/webhook-url/${platform}`);
           const data = await res.json();
           document.getElementById('webhookUrlInput').value = data.webhook_url;
       }
       ```
     * Trigger this function on platform selection change.

3. **Deployment Wizard Behavior**
   - The Webhook URL field should always populate with the correct, complete URL:
     * Telegram: `https://<server>/webhook/telegram`
     * WhatsApp: `https://<server>/webhook/whatsapp`
     * Facebook: `https://<server>/webhook/facebook`
     * Instagram: `https://<server>/webhook/instagram`
   - Remove the need for manual entry unless the user wants to override.

4. **Test**
   - Select Telegram in the wizard and verify the field shows the full URL.
   - Do the same for WhatsApp, Facebook, and Instagram.
   - Confirm the URL matches the current Replit public URL automatically.
